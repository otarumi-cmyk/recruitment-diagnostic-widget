# 人気300企業 転職可能性診断（完全版）

=========================================

**目的：**  
あなたの市場価値を100点満点でスコア化し、人気企業300社への「合格可能性（％）」を算出します。

---

-----------------------------------------

## ◆ 診断フォーム（20問）

-----------------------------------------

以下の20問に回答するだけで、市場価値スコアを算出できます。すべて選択式で点数化済み。

### UI要件（質問画面）
- フォーム形式: 単一選択/複数選択（maxSelectionsあり）を明示。未選択時はエラー表示。
- 進行状況: トップにプログレスバーと「回答数/総数（%）」を表示。
- 入力操作: 選択肢はボタン型（カード/ピル）。選択状態を色で強調（プライマリ色）。
- ナビゲーション: 「戻る」「次へ」ボタン。最終設問では「結果を見る」。
- モバイル対応: 幅600px未満では選択肢を縦並び、ボタン幅100%。
- バリデーション: 未選択で次へを押したら、その設問にエラーメッセージ。
- スキップ/分岐: 条件付き設問は条件を満たさない場合スキップし、進行数は自動計算。
- 配色/トーン: プライマリ #f54b02, セカンダリ #ff6b03 を基本とし、背景白、テキスト黒系。
- 操作性: クリック/タップ全域をヒットエリア化（padding確保、cursor: pointer）。
- アクセシビリティ: 視覚的な選択状態＋フォーカスアウトライン（実装時）、ラベル文言は明確・簡潔。

## ◆ スコア計算アルゴリズム

=========================================

### Step1：20問の点数を集計

```
S_exp = Q1 + Q2 + Q3 + Q4 + Q5 + Q6
S_income = Q7 + Q8 + Q9
S_skill = Q10 + Q11 + Q12
S_fit = Q13 + Q14 + Q15
S_academic = Q16 + Q17
S_person = Q18 + Q19 + Q20
```

### Step2：100点に正規化して市場価値スコア化

```
職務経験 = (S_exp / 50) * 40
年収役職 = (S_income / 23) * 20
スキル汎用性 = (S_skill / 15) * 15
フィット度 = (S_fit / 30) * 15
学歴資格 = (S_academic / 7) * 5
人物特性 = (S_person / 15) * 5
```

### Step3：市場価値スコア（0〜100）

```
市場価値 = 
職務経験 + 年収役職 + スキル汎用性 +
フィット度 + 学歴資格 + 人物特性
```

## ◆ 合格可能性アルゴリズム

=========================================

### Step1：企業ランクの設定（必要市場価値）

| ランク | 必要市場価値 |
| --- | --- |
| A | 80 |
| B | 70 |
| C | 60 |
| D | 50 |
| E | 40 |
| F | 30 |

（300社すべてに付与）

### Step2：入社難易度の補正（難易度係数 k_diff）

| 難易度ランク | k_diff |
| --- | --- |
| A | 1.15 |
| B | 1.10 |
| C | 1.05 |
| D | 1.00 |
| E | 0.95 |
| F | 0.90 |

- 実力主義・高倍率ほど A / B を付与し、必要市場価値を実質的に引き上げるイメージ
- 安定志向・母集団広めの企業は D〜F で緩和

### Step3：合格可能性の計算式（95%が1〜6%に収まる設定）

```
gap = あなたの市場価値 - (必要市場価値 × k_diff)
p = 0.01 + 0.05 * sigmoid((gap - g0) / s)
  where sigmoid(x) = 1 / (1 + exp(-x))

推奨パラメータ: g0 = 0, s = 4
合格確率(%) = clip(p, 0.01, 0.06) × 100
```

- gap=0 で約3.5%、gap がプラスでも最大6%、マイナスでも最小1%
- gap が ±12〜15pt 程度の現実的レンジなら 95% 以上が 1〜6% に収まる
- 分布を上下させたい場合は g0 を微調整（+で全体を下げ、-で上げる）
- ばらつきを変えたい場合は s を変更（大きいほど勾配がゆるくなる）

### Step4：視覚化（ASCIIメーター仕様）

- 20マスのメーターを表示し、6% をフルスケールとして割合表示  
  `filled = round(prob% / 6 * 20)`  
  `bar = "█" * filled + "░" * (20 - filled)`
- 色分け（テキストラベルで表現、UI側でCSS/SVGに置換可）
  - GREEN  : 6%以上
  - YELLOW : 4%以上 6%未満
  - ORANGE : 2%以上 4%未満
  - RED    : 1%以上 2%未満（下限1%でクリップ）
- 表示例（テキスト）
